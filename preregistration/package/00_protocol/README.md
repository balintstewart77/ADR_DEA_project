# Protocol

This folder contains the reviewed protocol source candidates and, after
approval, will contain the final registration PDF. It must not contain coder
assignments, contact details, or post-registration amendments.

`Validation_Protocol_PreReg_v0.12.docx` is the current protocol implementation
and collaborator-review candidate. It records the post-pilot shared calibration,
qualitative model-direction check, closed coder-feedback process, and current
diagnostic-instrument status. It is prospective with respect to the formal validation sample
and formal coding, but it is not the final frozen or registered protocol. The
preceding v0.11, v0.10 and v0.9 candidates are preserved as superseded,
unregistered history.

Protocol v0.12 was amended in place, under its unregistered working-candidate
status, to record simultaneous circulation on 21 July 2026, closure of coder
feedback, and the subsequent candidate-0.7 governance correction. Candidate
0.6 was imported into the live scratch project and partially inspected, then
superseded before final runtime QA; it collected no formal data. Candidate 0.7
is now the current formal-instrument review candidate and is not frozen or
live-runtime-QA approved.

The active protocol now specifies one permitted-material declaration per
scratch coder before formal coding and a separate exposure question for every
coder–project assignment. Exposure covers prior involvement, professional or
institutional familiarity, and accidental reviewer-information exposure. A
flagged coder still classifies from permitted evidence; flagged responses remain
in the primary analysis, with conditional exclusion sensitivity analysis where
informative. Detailed instrument history remains in the REDCap version history
rather than protocol Section 6.2. The same working candidate records that the
model-direction check was reverified against the frozen canonical 1,308-row
GPT-5.5 artefact before preregistration.

Final freeze remains conditional on final formal-instrument live QA, Jo's final substantive review,
propagation of resulting changes, and final package QA. No official active or
reserve sample draw and no formal coding are authorised under this status. The canonical
machine-readable status and exact protocol provenance are in
`../../preregistration_artifact_manifest.csv`.
